Assignment 1: Data Collection and Analysis

Approach: We analyzed elevator arrival data to calculate the probability of each elevator arriving next, based on their average frequencies. Using these probabilities, a weighted average of the elevator coordinates is computed to find the optimal waiting position that minimizes walking distance. Finally, the effectiveness of this "smart" position is evaluated by comparing the average walking distances from both the initial guess and the calculated position on test data.

Assignment 2: KMeans Clustering Visualization Webpage

Approach: This project is a full-stack web application for visualizing KMeans clustering, built using Python (Flask) for the backend, and JavaScript for the interactive front-end. Flask serves the HTML, CSS, and JavaScript files, and can handle backend logic. Users can generate datasets, select initialization methods (Random, Farthest First, KMeans++, or Manual), and watch the clustering process step-by-step or to full convergence. The front-end, implemented with JavaScript, handles dataset creation, centroid placement, and cluster assignment, while CSS is used to style the interface. The entire KMeans algorithm is implemented from scratch using JavaScript.

Assignment 3: SVD Preprocessing on MNIST with Logistic Regression

Approach: In this assignment, we applied Singular Value Decomposition (SVD) to the MNIST dataset for dimensionality reduction, followed by training a logistic regression model. By experimenting with different numbers of singular components, we explored how reducing the dataset's dimensionality impacts model performance. The analysis focused on the trade-off between accuracy and training time, with diminishing returns observed after retaining around 200 components. We also visualized the top singular vectors to better understand the dominant patterns captured in the data. Logistic regression, while simple, proved to be effective, though it showed limitations in reaching perfect accuracy.
